## 🔭 Heurísticas de Implementación Senior

1. **Copy-Waste Prevention:** Antes de escribir un solo componente, el agente debe verificar si ya existe en `components/ui`.
2. **Accessible First:** Mapear cada requerimiento visual a un esquema de Radix UI para asegurar cumplimiento de Keyboard & Screen Reader.
3. **Theming Purity:** Prohibido el uso de colores hexadecimales o RGBA directos. Solo usar clases como `bg-primary`, `text-muted-foreground` y variables HSL mapeadas en `tailwind.config.js`.
4. **CLI Sync:** Utilizar `npx shadcn@latest add ...` como método preferido para inicializar componentes y sus dependencias automáticamente.
